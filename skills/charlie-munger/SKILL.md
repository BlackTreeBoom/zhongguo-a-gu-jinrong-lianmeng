# 查理芒格框架

这个 skill 是整个仓库里最底层的方法论 skill。它适合拿来校正思考方式，而不是替代行业研究。

## 何时使用

- 用户要做重大投资决策
- 用户在多个看起来都不错的机会之间犹豫
- 用户需要建立一套可迁移的判断原则
- 用户要降低犯大错的概率

## 先做什么

先读 `references/profile.md`。如果要给出处，再读 `references/sources.md`。

## 分析顺序

1. 先反过来想。
   不是问“怎么赢”，先问“怎样会死得很惨”。
2. 再画能力圈。
   哪些部分你真的懂，哪些只是听说过。
3. 再调用多学科模型。
   经济学、心理学、博弈、激励、会计、概率、规模效应，要交叉看。
4. 再做清单式排错。
   激励错配、杠杆、周期误判、数据幻觉、确认偏误等，逐项排查。
5. 最后才谈下注大小。
   真正的高把握机会不多，但出现时应敢于集中。

## 输出格式

- `反向清单`
- `能力圈`
- `关键思维模型`
- `激励与偏误`
- `最可能的大错`
- `结论`

## 风格要求

- 冷静、克制、少废话
- 不神化复杂模型
- 优先避免愚蠢，而不是追求聪明
- 把判断建立在常识和跨学科联系上

## 不该怎么用

- 不要把它写成鸡汤
- 不要只列名言，不落到具体判断
- 不要无视激励机制和人性偏差

# 雪球金融街行藏框架

这个 skill 适合做“信息密度高、事件变化快、需要拆文本和路径”的投资分析。

## 何时使用

- 用户要研究政策、法案、监管文本对公司影响
- 用户需要看医药外部风险、海外法案、供需与产业链事件
- 用户要在多个机会之间比较确定性和赔率

## 先做什么

先读 `references/profile.md`。如果要引用来源，再读 `references/sources.md`。

## 分析顺序

1. 先列事件事实。
   时间点、主体、流程、下一节点，别只给情绪判断。
2. 再列市场误读点。
   哪些是市场放大的，哪些是文本里真正写了的。
3. 再推演路径。
   最乐观、基准、最差三种情况分别怎样。
4. 再比较赔率与确定性。
   不是只问能涨多少，而是问胜率是否足够。
5. 最后才给操作倾向。
   更像研究员式结论，而不是喊单。

## 输出格式

- `事件概览`
- `文本/信息要点`
- `市场误读`
- `情景推演`
- `赔率与确定性`
- `结论`

## 风格要求

- 证据链优先
- 尽量还原时间线
- 避免一句话盖棺定论
- 宁可写慢一点，也不要跳步骤

## 不该怎么用

- 不要脱离文本和事实硬猜
- 不要把它写成鸡血文
- 不要把“有观点”误当成“有证据”
